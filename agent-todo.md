## CONTENT GUIDELINES

### For All New Topics
- Follow the format: Introduction → Problem → Why it Exists → Real-world Analogy → Definition → Key Concepts → Diagrams (Mermaid) → Code Examples (Python/Java) → Interview Questions → Best Practices → Related Topics
- Include pros/cons, common mistakes, and when NOT to use
- Keep code examples minimal and production-ready
- Cross-link related topics using relative paths

### For Design Patterns
- Include Intent, Motivation, Structure, Participants, Collaboration, Implementation, and Known Uses
- Show UML diagrams where applicable
- Provide real-world examples from popular libraries/frameworks

### For HLD Case Studies
- Include: Functional Requirements, Non-Functional Requirements, Capacity Estimation, System APIs, Database Design, Caching Strategy, Scaling Strategy, Bottlenecks, Trade-offs, Failure Handling, Monitoring & Metrics, Deployment Strategy

### For LLD Machine Coding
- Include: Problem Statement, Requirements, Class Diagram, Implementation, Test Cases, Edge Cases, Complexity Analysis
