# Concurrency in Python

This section covers the concurrency models, primitives, and frameworks in Python. Developers must understand when to apply multi-threading, multi-processing, and asynchronous execution (asyncio) depending on whether tasks are CPU-bound or I/O-bound.

## Topics

- [Threads & GIL](/03-lld/concurrency-python/threads-gil): Standard multithreading, mutexes, variables, and the CPython Global Interpreter Lock.
- [Multiprocessing](/03-lld/concurrency-python/multiprocessing): True parallel execution, Inter-Process Communication (IPC), and shared state.
- [Asyncio & Coroutines](/03-lld/concurrency-python/asyncio-coroutines): Event loops, coroutines, async/await, and cooperative multitasking.
- [Pools & Futures](/03-lld/concurrency-python/pools-futures): Thread/Process pools, execution management, and future callbacks.
