# SDE Interview Preparation Knowledge Base

Welcome to the **SDE Interview Prep Platform**! This repository is your comprehensive, open-source companion for cracking software engineering interviews. It is structured as a book-like learning path covering everything from core data structures to advanced system architecture.

Whether you are preparing for a mid-level Backend Engineer role or a Staff-level System Architecture interview, this guide provides the foundational theory, real-world case studies, and code implementations you need to succeed.

---

## 🗺️ The Learning Path

We recommend tackling the topics in the following order to build a solid foundation before diving into complex systems:

### 1. ⚡ [DSA Patterns](./04-dsa-patterns)
Master the core algorithms and data structure patterns (Sliding Window, Two Pointers, Dynamic Programming, etc.) essential for the coding rounds.

### 2. 🎨 [Design Patterns](./01-design-patterns)
Understand the Gang of Four (GoF) design patterns—Creational, Structural, and Behavioral—to write clean, extensible, and maintainable code.

### 3. 🧩 [Low-Level Design (LLD)](./03-lld)
Dive into Object-Oriented Programming (OOP) fundamentals, SOLID principles, UML modeling, and machine coding problems (e.g., designing an ATM or Parking Lot).

### 4. 🏗️ [High-Level Design (HLD)](./02-hld)
Learn how to design scalable, fault-tolerant distributed systems. Covers databases, caching, microservices, and in-depth case studies of real-world apps like Netflix, Uber, and WhatsApp.

### 5. 🔄 [Concurrency & OS](./05-concurrency-and-os)
Grasp the intricacies of multithreading, synchronization, memory models, and OS fundamentals necessary for writing highly performant code.

### 6. 🗣️ [Behavioral & Leadership](./06-behavioral-and-leadership)
Prepare for the "soft skills" rounds by mastering stakeholder management, decision-making, and team leadership communication frameworks.

## 🎯 Platform Goals

- **Comprehensive coverage:** From basic arrays to Paxos consensus protocols.
- **Practical applications:** Focus on real-world analogies, production-grade architectural patterns, and practical trade-offs.
- **Visual learning:** Utilize Mermaid diagrams for architecture flows and schemas.
- **Interview-focused:** Dedicated Q&A sections ranging from Beginner to Staff Engineer levels.

## 🤝 How to Contribute

This platform thrives on community contributions. If you'd like to add a new topic or fix a typo:
1. Please read our [Contributing Guidelines](./contributing) first.
2. Create new topic folders under the appropriate `src/pages/<section>/` directory.
3. Ensure every concept uses its own `index.mdx` file.
4. Keep the content concise, interview-focused, and beginner-friendly.
