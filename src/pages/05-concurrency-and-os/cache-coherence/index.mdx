# Cache Coherence

## Introduction
Cache coherence ensures that multiple CPU caches see a consistent view of shared memory.

## Problem Statement
When multiple cores cache the same memory location, stale values can lead to incorrect behavior.

## Why this exists
Modern processors use local caches for performance, but those caches must stay coherent when data is shared.

## Key concepts
- **Write-back cache**
- **Cache line**
- **Invalidation**
- **Coherence protocol**
- **MESI**

## Best practices
- Minimize false sharing by aligning shared data carefully.
- Prefer immutable or thread-local data for concurrent workloads.
- Use atomic operations when sharing cache-coherent state.
