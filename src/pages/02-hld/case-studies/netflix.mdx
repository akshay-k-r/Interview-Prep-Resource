# Netflix (Subscription Streaming Service)

## Introduction
Netflix is a global streaming service that accounts for a massive percentage of total global internet bandwidth. While similar to YouTube in streaming mechanics, Netflix has a fundamentally different architecture because it is a "read-heavy, controlled-write" system. Netflix encodes its own professional content rather than dealing with millions of unpredictable user uploads.

## Problem Statement
Netflix must serve ultra-high-quality (4K Dolby Vision) video to 200+ million subscribers simultaneously without buffering. If AWS goes down in one region, the system must failover to another region seamlessly.

## Functional Requirements
1. Users can create profiles and manage subscriptions.
2. Users can browse personalized recommendations.
3. Users can stream videos seamlessly across different devices (TV, Mobile, Web).
4. System must save playback state (resume watching).

## Non-Functional Requirements
1. **Zero Downtime:** High availability is the top priority.
2. **Low Latency Streaming:** No buffering.
3. **High Security:** DRM (Digital Rights Management) to prevent piracy.

## Core Architecture: The Three Main Components
Netflix's architecture is divided into three distinct parts:
1. **Client (App/TV):** The device playing the video.
2. **Control Plane (Backend):** Runs on AWS. Handles login, billing, recommendations, search, and telling the client *where* to get the video.
3. **Data Plane (Open Connect):** Netflix's proprietary CDN. Handles the actual heavy lifting of streaming the video files.

## Step-by-step Playback Flow
1. **Browse:** The user opens the app. The app talks to the Control Plane (AWS) via the API Gateway. AWS fetches personalized recommendations and movie metadata.
2. **Play Request:** The user clicks "Play" on *Stranger Things*.
3. **Routing:** The AWS backend analyzes the user's IP address and ISP. It queries the Open Connect routing service and returns a list of 3-5 specific Open Connect Appliance (OCA) IP addresses that are physically closest to the user and hold the video file.
4. **Streaming:** The Netflix app disconnects from AWS and connects directly to the OCA server sitting inside the user's local ISP (e.g., Comcast or AT&T). The video streams directly from the local ISP node to the TV using adaptive bitrate streaming (DASH).

## Open Connect (The Secret Weapon)
Why doesn't Netflix just use Amazon CloudFront or Akamai? 
Because standard CDNs charge per gigabyte. Given Netflix's volume, standard CDN costs would bankrupt them. 

Instead, Netflix built **Open Connect**. They manufacture custom, high-density storage servers (Open Connect Appliances) and literally ship them for free in boxes to ISPs all around the world. The ISP plugs the server into their local network.
- *Result:* When you watch Netflix, the video file never travels across the backbone of the internet. It travels from a box sitting in your local town directly to your house. This saves the ISP money on bandwidth and guarantees zero buffering for the user.

## Database Design (Cassandra)
Netflix relies heavily on **Apache Cassandra** (Wide-Column NoSQL) for its Control Plane because it requires high availability, cross-region replication, and no single point of failure.

**Handling Playback State (Bookmarks):**
- When a user watches a movie, the app sends a heartbeat every 5 seconds to AWS: "User 123 is at minute 42."
- This is a massive write load. Cassandra handles this beautifully. 
- If the user switches from TV to Mobile, the Mobile app queries Cassandra, gets "minute 42", and resumes playback instantly.

## Scaling Strategy & Resilience
- **Chaos Engineering (Chaos Monkey):** Netflix intentionally shuts down production servers, kills databases, and cuts network cords during business hours to ensure their system automatically heals and fails over without users noticing.
- **Multi-Region Active-Active:** The AWS backend runs simultaneously in US-East, US-West, and EU. If AWS US-East goes down entirely, Netflix uses DNS routing to instantly flip all East Coast users to US-West. Users might experience a 5-second lag, but streaming continues.

## Bottlenecks & Trade-offs
- **Storage vs Network:** Netflix encodes a single movie into dozens of different formats and bitrates, generating terabytes of files for one movie. They push *all* these files to the OCAs overnight when internet traffic is low. Trade-off: Massive redundant storage costs at the edge, but it entirely eliminates network congestion during prime-time viewing hours.

## Summary
Netflix is the gold standard of cloud architecture and edge delivery. By utilizing a highly resilient AWS microservices backend for the Control Plane, and inventing a proprietary, ISP-embedded CDN (Open Connect) for the Data Plane, Netflix achieves unparalleled streaming quality and reliability.
