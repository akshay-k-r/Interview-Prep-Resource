# Structural Patterns

Structural patterns deal with object composition, creating relationships between entities to form larger structures.

## Patterns

- [Adapter](./adapter)
- [Decorator](./decorator)
- [Facade](./facade)
- [Flyweight](./flyweight)
- [Proxy](./proxy)
- [Bridge](./bridge)
- [Composite](./composite)

## Key Principles

- Compose objects into complex structures while keeping client code simple.
- Decouple abstraction from implementation.
- Use composition over inheritance where appropriate.
